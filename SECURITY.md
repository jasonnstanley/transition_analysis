# Security Policy

## Supported Versions

Security updates will be applied to the latest development version of the repository.

| Version | Supported |
|---------|-----------|
| Latest release | ✓ |
| Older releases | ✗ |

## Reporting a Vulnerability

If you believe you have discovered a security issue in this repository, please report it privately before making it public.

Please include:

- a description of the issue
- steps to reproduce the problem
- the potential impact
- any suggested mitigation

Security reports will be acknowledged and investigated as promptly as practical.

## Scope

This repository contains research software and documentation.

No production service is operated from this repository.

No personally identifiable research data should be committed to the repository.

Approved public datasets are de-identified before publication.

## Dependency Management

Dependencies are maintained through the project's documented build process.

Contributors are encouraged to report outdated or vulnerable dependencies through a GitHub Issue or Pull Request.

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing security issues.
